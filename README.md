# IFRS 9 Credit Risk Modeling – PD, LGD, EAD & ECL

## Overview
This project presents an end-to-end **IFRS 9 credit risk modeling framework** implemented in Python.
It covers the full modeling lifecycle used in banking environments, from Probability of Default (PD)
to Expected Credit Loss (ECL), following regulatory best practices.

## Objectives
- Model **Probability of Default (PD)**
- Estimate **Loss Given Default (LGD)** and **Exposure at Default (EAD)**
- Compute **Expected Credit Loss (ECL)** under IFRS 9
- Incorporate **macroeconomic scenarios**
- Perform **model validation and backtesting**

## Methodology
- Data preparation and exploratory analysis
- PD modeling (logistic regression, survival analysis)
- LGD and EAD modeling
- IFRS 9 staging (Stage 1 / Stage 2)
- ECL calculation
- Model validation
